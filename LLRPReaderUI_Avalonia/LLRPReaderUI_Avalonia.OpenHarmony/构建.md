# LLRPReaderUI_Avalonia 套 OpenHarmony 壳（OHOS）

参考项目：
- https://github.com/OpenHarmony-NET/OpenHarmony.Avalonia

## 目标

把现有 Avalonia UI（`LLRPReaderUI_Avalonia/LLRPReaderUI_Avalonia`）跑在 OpenHarmony 上。

核心思路：
- UI 逻辑继续放在现有 Avalonia 主项目里。
- 新增一个 OpenHarmony 宿主（壳）工程，负责 `XComponent` + NAPI + 原生入口。
- 壳工程里通过 `Avalonia.OpenHarmony` 初始化并挂载 `App`。

## 你当前项目的好消息

`App.axaml.cs` 里已经有：
- `ISingleViewApplicationLifetime` 分支。

这正是移动端/嵌入式平台（含 OpenHarmony）所需要的生命周期，说明 UI 层基本可复用。

## 推荐路径（最快落地）

## 1. 先跑官方示例（验证工具链）

1. 拉取 `OpenHarmony.Avalonia`。
2. 按其文档在 DevEco Studio 打开 `OHOS_Project`。
3. 能在设备或模拟器跑起来后，再迁移你的 UI。

这样可以先排除 DevEco / SDK / 签名 / 设备权限问题。

## 2. 在你的仓库建立 OpenHarmony 壳目录

在 `LLRPReaderUI_Avalonia/LLRPReaderUI_Avalonia.OpenHarmony/` 下准备两部分：

1. `DotNetEntry/`（C# 入口项目）
2. `OHOS_Project/`（DevEco 工程）

建议先从官方仓库复制对应目录作为初始模板，再改命名与引用。

## 3. C# 入口项目（DotNetEntry）怎么接你的 App

入口关键点：

1. 依赖：
	- `Avalonia.OpenHarmony`
	- `OpenHarmony.NDK.Bindings`（按官方示例版本）
	- 你的 UI 项目引用：`..\LLRPReaderUI_Avalonia\LLRPReaderUI_Avalonia.csproj`

2. 在 `XComponent` 创建时实例化：
	- `new AvaloniaXComponent<App>(...)`

3. 确保导出入口函数：
	- `RegisterEntryModule`（`UnmanagedCallersOnly`）

4. AppBuilder 走 OpenHarmony 平台初始化：
	- `AppBuilder.Configure<App>().UseOpenHarmony()`

官方示例里这几块分别在 `Entry/napi_init.cs`、`Entry/XComponentEntry.cs`、`Avalonia.OpenHarmony` 平台扩展中。

## 4. OHOS 壳工程（DevEco）关键文件

壳工程里至少要有：

1. `entry/src/main/cpp/napi_init.cpp`
	- 通过 `dlopen("libavalonia.so")`
	- `dlsym("RegisterEntryModule")`
	- 调用 C# 导出入口。

2. `entry/src/main/cpp/CMakeLists.txt`
	- 编译并链接 `entry` native so。

3. `entry/src/main/ets/...`
	- 页面中放 `XComponent` 容器。

4. `module.json5`、`build-profile.json5`、签名配置
	- 按 DevEco 默认流程配置即可。

## 5. 产物放置关系（最容易踩坑）

需要让 OHOS 壳工程在运行时能找到 C# 编译出的共享库：

1. C# 侧发布得到 `libavalonia.so`（以及依赖 so）。
2. 将这些 so 放入 OHOS 工程 `entry` 模块对应 ABI 的 `libs` 目录。
3. 保证 `napi_init.cpp` 里 `dlopen` 的库名与实际产物一致。

只要这一步不通，应用会在 native 初始化阶段直接失败。

## 6. 你这个项目需要额外关注

1. `LLRPSdk` 网络/设备访问权限：
	- 在 OHOS 侧声明网络权限。

2. 文件路径：
	- 现在日志配置 `LoggingConfig.json` 在桌面按文件路径读取。
	- 到 OHOS 需要改成应用沙箱路径或内嵌资源读取。

3. 字体：
	- 你用了 `WithInterFont()`。
	- 在 OHOS 可先用系统字体验证，再决定是否打包自定义字体。

## 最小迁移顺序（建议按这个来）

1. 官方示例跑通。
2. 把示例里的 `App` 替换为你的 `LLRPReaderUI_Avalonia.App`。
3. 先渲染一个简单 `MainView`（不接 LLRP）。
4. 再逐步接回日志、配置、网络、设备通信。

## 常见问题

1. 黑屏但不崩溃：
	- 先查 `XComponent` 是否创建成功。
	- 再查 `OnSurfaceCreated` 是否触发。

2. 直接崩在启动：
	- 多半是 `dlopen` 找不到 `libavalonia.so` 或符号名不匹配。

3. 输入法/触摸异常：
	- 先对齐官方仓库当前分支代码，不要自行删改 `Entry` 初始化逻辑。

## 结论

你这个 Avalonia UI 可以直接套 OHOS 壳，且改动主要在宿主层，不在 UI 层。

建议先复用官方 `OHOS_Project + Entry` 模板，再把 `AvaloniaXComponent<TApp>` 的泛型 `TApp` 对到你现有 `App`，这是成功率最高的做法。
